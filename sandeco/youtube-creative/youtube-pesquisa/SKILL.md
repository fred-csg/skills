# YouTube Creative Research

## Overview
Esta skill transforma pesquisas genéricas em **bases estratégicas de conteúdo**. O objetivo não é apenas resumir fatos, mas encontrar ângulos de retenção, ganchos psicológicos e dados de autoridade para roteiros de alta performance no YouTube.

## Quando usar
- Ao iniciar a pesquisa para um novo vídeo do YouTube.
- Quando o usuário pede para "estudar um tema" ou "buscar referências" para criar conteúdos em vídeo.
- **NÃO use** para pesquisas acadêmicas puras, relatórios jurídicos ou técnicos sem a intenção de transformar em conteúdo voltado para audiência.

## Estrutura Obrigatória da Base de Pesquisa (The Master Doc)

Toda pesquisa gerada por esta skill DEVE conter as seguintes seções estruturadas:

1.  **O Gancho de Curiosidade (The Hooks):**
    - Identifique 3 ângulos de entrada: **Curiosidade Pura**, **Medo/Urgência** (FOMO) e **Contradição/Polemica** (ex: "Por que o que você sabe sobre X está obsoleto").
2.  **Dados de Autoridade e Prova Real:**
    - Nomes de empresas, especialistas, datas de lançamentos ou estatísticas reais. Evite termos vagos como "especialistas dizem". Cite nomes e fontes específicas.
    - **Links das Fontes:** Liste os links (URLs) das fontes mais importantes, notícias originais ou documentações técnicas encontradas. Toda afirmação de peso deve ter um link de referência.
3.  **Conflitos e Debates (Engagement Fuel):**
    - Encontre pontos de discórdia na comunidade. O que gera discussão nos comentários? (A vs B, Prós vs Contras acentuados).
4.  **Analogias Facilitadoras (Brain Anchors):**
    - Crie analogias simples para conceitos complexos. O espectador deve "entender visualmente" a explicação através da sua pesquisa.
5.  **Referências Visuais e de Estética:**
    - Sugestões técnicas de B-roll, estilos de edição (ex: Dark Mode, Minimalist, Cyberpunk) e exemplos de canais que usam estéticas similares.

## Processo de Pesquisa Deep-Dive (Workflow)

**Passo 0: Fase de Refinamento (Obrigatório)**
Antes de iniciar a navegação, você DEVE fazer 3 perguntas estratégicas ao usuário para calibrar a bússola da pesquisa:
1.  **Nível de Profundidade:** "O vídeo é para iniciantes (mais didático) ou experts (mais técnico/novidades de nicho)?"
2.  **Abordagem Primária:** "Qual o sentimento principal? (Otimismo/Futurismo, Alerta/Crítica ou Tutorial prático)?"
3.  **Diferencial do Canal:** "Existe alguma opinião ou 'tese' sua que eu deva reforçar ou usar para buscar provas durante a pesquisa?"

*Aguarde as respostas do usuário antes de seguir para o Passo 1.*

**Passo 1: Triangulação de Fontes**
- Busque em: Documentações oficiais + Opiniões de especialistas no Twitter/Reddit + Notícias recentes de veículos de tecnologia/negócios.
- Identifique a "temperatura" do assunto: O tema está em alta ou é uma dor latente mal resolvida?

**Passo 2: Extração de Dores e Desejos (Avatar Focus)**
- Para quem é o vídeo? O que essa pessoa perde se não souber disso? O que ela ganha se aprender?

**Passo 3: Storytelling Data**
- Transforme os fatos em uma jornada. A pesquisa deve mostrar uma evolução (Antes -> Mudança -> Depois).

## Common Mistakes (O que evitar)
- **Wikipedia-Style:** Não entregue uma lista monótona de fatos. A pesquisa deve ter um "tom" de descoberta.
- **Vagueza Digital:** Falar sobre "IA" sem citar qual modelo (GPT-4o, Claude 3.5, Gemini 1.5). 
- **Falta de Contexto Local:** Se o alvo é o Brasil, traga impactos ou dados relevantes para a realidade local se disponíveis.

## Red Flags - PARE e Refaça
- Se a pesquisa não contém nenhum "fato surpreendente".
- Se os ganchos (hooks) parecem manchetes de jornal antigo.
- Se não houver referências visuais descritas.

## Exemplo de Aplicação

**Input:** "Pesquise sobre WebMCP."
**Output esperado:**
> ### Base de Pesquisa: WebMCP - O Protocolo que Mata o Context Window Limit
> **Curiosidade (Hook):** "Esqueça o 'copy-paste' de logs. O WebMCP dá olhos e mãos reais para a sua IA."
> **Autoridade:** Anunciado na conferência X pelo Google DeepMind...
> **Diferencial:** Diferente do MCP da Anthropic que exige servidor local, o WebMCP usa recursos do browser...
> **Estética:** Visual de terminal moderno mesclado com animações de rede em vidro (glassmorphism).
